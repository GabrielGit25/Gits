ou remover e adicionar de novo

Resolução de falha
Commit para adicionar novo arquivo ao github ou atualizar

1. Confirmar qual branch você está agora:
git branch


2. Renomear o branch local para main ou correspondente (recomendado, igual ao GitHub)
git branch -M main
git push -u origin main


Possibilidade de falha:
O Git está dizendo que o repositório remoto (GitHub) já tem commits que você não tem localmente.

Opção 1 – Trazer o histórico do remoto e unificar (merge)
git pull origin main --allow-unrelated-histories
git push origin main

Opção 2 – Forçar push, sobrescrevendo o remoto (use só se não se importar de perder o que está no GitHub)
Isso apaga qualquer conteúdo que exista no branch main do GitHub e substitui pelo seu commit local. Faça só se tiver certeza de que não precisa do que está lá.
git push -f origin main

Atualizar mudanças de arquivos existentes localmente para o github

1. Inicializou um novo repositório Git na pasta do seu projeto.
 git init

2. git add .
Adicionar arquivos para à área de preparação (stage), prontos para serem salvos.

3. Salvar as mudanças localmente com uma mensagem descrevendo o que foi feito.
git commit -m "mensagem"

4. Conectar repositório local ao repositório remoto no GitHub.
git remote add origin https://github.com/GabrielGit25/projeto.git

5. git branch -M main

6. git push -u origin main --force

Commit para atualizar arquivos existentes no github

# 1) Verifique o status e a branch atual
git status
git branch --show-current

# 2) Puxe mudanças remotas antes (evita conflitos)
git pull --rebase origin main

# 3) Faça alterações no arquivo (ex.: src/index.ts)

# 4) Adicione ao stage
git add caminho/para/arquivo.ext
# ou para interativo: git add -p
# ou para tudo: git add .

# 5) Crie o commit
git commit -m "Mensagem descrevendo a alteração"

# 6) Envie para o GitHub
git push origin main
# se for a primeira vez desta branch no remote: git push -u origin main
